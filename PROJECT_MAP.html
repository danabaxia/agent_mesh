2. Delegation outcome → A2A Task status flow

Every message/send always returns a Task — outcomes are data, never JSON-RPC errors (except un-parseable requests). This maps each internal delegateTask result to its terminal A2A status.state (PROJECT.md §1.7).

completed failed rejected 
stateDiagram-v2
    [*] --> Received: message/send
    Received --> Validate

    Validate --> rejected_bad: params shape-invalid
    Validate --> Guards: valid {mode, task}

    Guards --> rejected_cycle: own root in call-path
    Guards --> rejected_depth: depth budget exhausted
    Guards --> rejected_ro: do from ask parent
    Guards --> Spawn: passes

    Spawn --> failed_spawn: spawn_failed
    Spawn --> Worker: claude -p (cwd=B)

    Worker --> completed: exit 0 (done)
    Worker --> failed_timeout: timeout -> tree-kill
    Worker --> failed_internal: non-zero exit

    state "completed" as completed
    state "failed (spawn_failed)" as failed_spawn
    state "failed (timeout)" as failed_timeout
    state "failed (internal)" as failed_internal
    state "rejected (bad_input)" as rejected_bad
    state "rejected (cycle)" as rejected_cycle
    state "rejected (depth_budget)" as rejected_depth
    state "rejected (readonly_parent)" as rejected_ro

    completed --> [*]
    failed_spawn --> [*]
    failed_timeout --> [*]
    failed_internal --> [*]
    rejected_bad --> [*]
    rejected_cycle --> [*]
    rejected_depth --> [*]
    rejected_ro --> [*]

Write-boundary denials are not a terminal state: the path-guard hook denies the write inside the worker (exit 2, logged, counted in isolation_violations); the task still finishes completed/failed with the write simply absent from files_changed. boundary_denied is reserved/planned.

3. Runtime delegation flow (one message/send)

What actually happens end-to-end when a caller delegates to a peer. The two mode branches (ask read-only vs do write+hook) are the security-critical fork.

flowchart TD
    U(["Caller (Agent A)"]) --> RG["registry.js
resolve peer → canonical root"]
    RG --> SP["stdio-client.js
spawn long-lived peer server"]
    SP --> MS["message/send (A2A)"]
    MS --> V{"validateMessageSendParams"}
    V -- invalid --> T1["Task: rejected (bad_input)"]
    V -- valid --> G{"enterCallContext
cycle / depth / readonly_parent"}
    G -- refuse --> T2["Task: rejected"]
    G -- ok --> Q["SerialQueue
(do = 1 writer per folder)"]
    Q --> CS["captureChangeState (git snapshot)"]
    CS --> MODE{"mode?"}

    MODE -- ask --> A1["read-only tools
+ read-only-marked MCP grant"]
    MODE -- do --> D1["read + write tools
+ PreToolUse path-guard hook
+ acceptEdits"]

    A1 --> SPAWN["process.js: spawn claude -p
cwd=B · detached"]
    D1 --> SPAWN
    SPAWN --> WORK["worker runs
(writes gated by hook)"]
    WORK --> TO{"timeout?"}
    TO -- yes --> KILL["tree-kill: SIGTERM → SIGKILL
await tree death, then resolve"]
    TO -- no --> DONE["worker exits"]
    KILL --> CD["computeFilesChanged
porcelain ∪ digest delta"]
    DONE --> CD
    CD --> LOG["writeRunLog (.agent-mesh/logs)"]
    LOG --> TASK["buildTaskFromDelegateResult"]
    TASK --> RET(["Task back to caller"])
